= 共有モードと仮想モードの選び方
:revdate: 2025-09-16
:page-revdate: {revdate}

このガイドは、仮想クラスターに適したモードを選ぶ手助けをします：*共有* または *仮想*。

不明な場合は、*共有モード*から始めてください - これはデフォルトであり、最も一般的なシナリオに適しています。

== 共有モード（デフォルト）

*最適な用途：*

* Kubernetesの内部を管理せずにワークロードを迅速に実行したい開発者
* すべてのワークロードに対する可視性と制御を必要とするプラットフォームチーム
* ホストレベルのリソースにアクセスする必要があるユーザー（例えば、GPU）

*共有モード*では、仮想クラスターは独自のK3sサーバーを実行しますが、ワークロードの実行にはホストに依存します。仮想kubeletはリソースを同期し、クラスターリソースの分離をサポートしながら、軽量で迅速なプロビジョニングを可能にします。link:./../architecture.adoc#shared-mode[アーキテクチャ]に関する詳細。

=== ペルソナ別のユースケース

==== 開発者

_"私は仮想クラスターの外部に公開されるべきウェブアプリを構築しています。"_ +
→ *共有モード*を使用してください。これにより、アプリケーションをxref:how-tos-for-user/expose-workloads.adoc[公開]できます。

==== データサイエンティスト：

_"`I need to run Jupyter notebooks that leverage the cluster's GPU.`"_ +
→ *共有モード*を使用してください。物理デバイスへのアクセスを提供しつつ、オーバーヘッドを低く保ちます。

==== プラットフォーム管理者

_"私は中央の場所からすべてのテナントのワークロードを監視し、保護したいです。"_ +
→ *共有モード*を使用してください。ホストレベルのエージェント（例えば、監視、ポリシーの強制）は、すべての仮想クラスターで機能します。

==== セキュリティエンジニア

_"私はすべてのワークロードに対してネットワークポリシーやランタイムスキャンのようなセキュリティポリシーを強制する必要があります。"_ +
→ *共有モード*を使用してください。プラットフォームは、テナントのバイパスなしにポリシーをグローバルに強制できます。

_"私は新しいアドミッションコントローラーまたはポリシーエンジンをテストする必要があります。"_ +
→ 仮想クラスター内に限定される場合は、*共有モード*を使用してください。ホストに影響を与えずに、Kubewardenのようなツールを実行できます。

==== CI/CDエンジニア

_"私はパイプラインの実行ごとに使い捨ての仮想クラスターを迅速に、低リソースコストで立ち上げたいです。"_ +
→ *共有モード*を使用してください。プロビジョニングが迅速で、短命のネームスペーススコープの環境に最適です。

== 仮想モード

*最適な用途：*

* 完全なKubernetesの隔離が必要な上級ユーザー
* 実験的またはクラスター全体の機能をテストしている開発者
* Kubernetesの制御プレーン全体を制御する必要があるユースケース

*仮想モード*では、仮想クラスターは独自の隔離されたKubernetes制御プレーンを実行します。異なるCNIとAPI設定をサポートし、--深い実験や高度なワークロードに理想的です。link:./../architecture.adoc#virtual-mode[アーキテクチャ]に関する詳細。

=== ペルソナ別のユースケース

==== 開発者

_"私はホストクラスターで無効になっている新しいKubernetes機能ゲートをテストする必要があります。"_ +
→ *仮想モード*を使用してください。自分自身のコントロールプレーンフラグとAPI機能を設定できます。

==== プラットフォーム管理者

_"Kubernetesのバージョン間でのアップグレードをテストしており、新しいAPIの動作を含んでいます。"_→仮想モードを使用してください。異なるKubernetesのバージョンを実行し、安全にアップグレードパスを検証できます。

==== ネットワークエンジニア

_"クラスターのネットワーキングを完全に制御する必要がある新しいCNIを評価しています。"_ +
→ *仮想モード*を使用してください。ホストや他のテナントに影響を与えずに、別のCNIスタックを実行できます。

==== セキュリティエンジニア

_"クラスター全体に展開する前に、新しいアドミッションコントローラーとポリシーエンジンをテストしています。"_ +
→ *仮想モード*を、クラスター全体のポリシー、カスタムアドミッションフロー、または完全な制御を伴う高度な拡張機能のテストが必要な場合に使用してください。

== 追加情報

より高度なユースケースを評価している場合や、より深い比較を希望する場合は、xref:../architecture.adoc[アーキテクチャのドキュメント]で完全なトレードオフの内訳を確認してください。
