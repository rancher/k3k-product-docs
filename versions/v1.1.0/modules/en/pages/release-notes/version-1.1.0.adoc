= SUSE Virtual Cluster v1.1.0 Release Highlights
:revdate: 2026-05-27
:page-revdate: {revdate}

== New Features

* **Enhanced Security & Execution**: Virtual clusters now support custom runtime classes and security contexts, as well as `hostUsers` mapping directly in the cluster spec. (link:https://github.com/rancher/k3k/pull/742[#742], link:https://github.com/rancher/k3k/pull/787[#787])
* **Storage Sync**: Host StorageClasses are now seamlessly synced to the virtual cluster. For more details on how this works, check out our new link:https://documentation.suse.com/cloudnative/suse-virtual-clusters/1.1.0/en/resource-sync.html[Resource Sync guide]. (link:https://github.com/rancher/k3k/pull/681[#681])
* **Resource & Scheduling Control**: Major upgrades to scheduling, including the introduction of Pod affinity (preferring scheduling Pods on the same Node), worker limits in shared clusters, and explicit server/worker resource requests and limits. (link:https://github.com/rancher/k3k/pull/696[#696], link:https://github.com/rancher/k3k/pull/724[#724], link:https://github.com/rancher/k3k/pull/798[#798], link:https://github.com/rancher/k3k/pull/832[#832])

== Core Improvements

* **Smarter Distribution**: Overhauled the distribution algorithm to accurately account for host capacity, properly removing node capacity from merged lists and filtering requested infrastructure resources. (link:https://github.com/rancher/k3k/pull/688[#688], link:https://github.com/rancher/k3k/pull/816[#816], link:https://github.com/rancher/k3k/pull/834[#834])
* **Architecture Optimizations**: Improved Shared Mode by removing the Pod mutating webhook, transitioning `ControllerReferences` over to `OwnerReferences` for cleaner garbage collection, and changing default node types to worker. (link:https://github.com/rancher/k3k/pull/659[#659], link:https://github.com/rancher/k3k/pull/662[#662], link:https://github.com/rancher/k3k/pull/699[#699], link:https://github.com/rancher/k3k/pull/774[#774])
* **Upgrade Detection**: Added a notification banner to the UI to prompt for K3k upgrades when a targeted cluster is running an older version of the Virtual Cluster engine. (UI link:https://github.com/rancher/virtual-clusters-ui/pull/138[#138])
* **Enhanced UX**: During virtual cluster creation within the UI, the content of the Virtual Cluster Policy is now available in a read-only view for easier verification. (UI link:https://github.com/rancher/virtual-clusters-ui/pull/122[#122])

== Bug Fixes & Security

* **Dependency & Security Updates**: Upgraded to `Go v1.25.9`, virtual-kubelet `v1.12`, Kubernetes dependencies to `v1.35`, and bumped the base image to BCI `v16`, pulling in the latest upstream security patches, telemetry updates, and fixes. (link:https://github.com/rancher/k3k/pull/716[#716], link:https://github.com/rancher/k3k/pull/759[#759], link:https://github.com/rancher/k3k/pull/819[#819])
* **Policy Handling Fixes**: Improved policy status and deletion handling by adding finalizers and tracking `policy` directly within the Cluster Status. (link:https://github.com/rancher/k3k/pull/663[#663], link:https://github.com/rancher/k3k/pull/781[#781])
* Added support for projected service account tokens in shared mode to ensure service accounts operate correctly. (link:https://github.com/rancher/k3k/pull/660[#660])
* Fixed cgroup directory mapping for virtual mode clusters and addressed missing network policies specifically on `kube-ovn` networks. (link:https://github.com/rancher/k3k/pull/691[#691], link:https://github.com/rancher/k3k/pull/792[#792])
* Addressed an issue with translated ingress TLS secrets in ingress objects. (link:https://github.com/rancher/k3k/pull/672[#672])
* Fixed a UI bug regarding quotas in the Virtual Cluster policy and added missing quota definitions. (UI link:https://github.com/rancher/virtual-clusters-ui/pull/135[#135])
* Improved error handling for cluster provisioning within the UI. (UI link:https://github.com/rancher/virtual-clusters-ui/pull/111[#111])
